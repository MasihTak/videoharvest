# Security Policy

Thank you for helping make **VideoHarvest** more secure.

We take security seriously and appreciate responsible disclosure of vulnerabilities. This document explains which versions are supported, how to report security issues, and what you can expect during the disclosure process.

----------
## Supported Versions
Security updates are provided for the latest stable release.

| Version                | Supported                |
|------------------------|--------------------------|
| Latest                 | ✅                        |
| Previous major release | ✅ (critical issues only) |
| Development (main)     | ⚠️ Best effort           |
| Older releases         | ❌                        |


If you discover a vulnerability in an unsupported version, please verify that it still exists in the latest release before reporting it.

----------
## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Email: masihtak@outlook.com

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

You will receive a response within 72 hours. If confirmed, a patch will be released as soon as possible.


## Third-Party Software

VideoHarvest relies on several open-source projects, including:

-   **yt-dlp**
-   **FFmpeg**
-   **Tauri**
-   **Vue.js**

If a vulnerability originates from one of these upstream projects, please report it to the respective maintainers as well.

----------

## Responsible Disclosure

We greatly appreciate researchers and community members who help improve the security of this project through responsible disclosure.

Your efforts help make VideoHarvest safer for everyone.

Thank you for helping keep the project secure.
